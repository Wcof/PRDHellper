# PRD 编写规范

## 1. PRD 类型

本项目区分两类 PRD：

1. 系统级 PRD：描述完整系统或完整模块，按业务需要组织章节结构。
2. 页面级 PRD：描述单个页面、路由、弹窗、抽屉或局部交互，不强行套系统级长文档结构。

## 2. 页面级 PRD 必须包含

- 页面基础信息
- 页面目标
- 页面结构
- 字段说明
- 操作说明
- 交互规则
- 状态流转
- 异常场景
- 权限规则
- 数据规则
- 验收标准
- 变更记录

## 3. 系统级与页面级 PRD 落盘规则

1. 系统级 PRD 统一写到 `docs/prd/system/`。
2. 页面级 PRD 统一写到 `docs/prd/pages/`。
3. 单个根目录总文档只能作为中间草稿或汇总视图，不能替代 `system/` 和 `pages/` 下的正式产物。
4. 如果系统级 PRD 已经明确列出页面、模式页、弹窗、抽屉、独立工作台或操作终端，必须同步拆出对应页面级 PRD。
5. 拆分信息不完整时，允许先生成页面级占位草稿，但不能什么都不落盘。

## 4. 信息不足处理

不能确认的内容统一写为：

```text
[TODO: 需要确认具体内容]
```

不要编造业务规则。

## 5. 页面代码与 PRD 同步

页面代码发生功能、字段、按钮、交互、状态、权限、数据逻辑、异常处理、路由入口变化时，必须同步更新页面 PRD。
