# 模板自定义与维护教程

> 目标：告诉你“应该改哪个文件夹”和“改完如何生效”。

## 1. 先区分两类修改

1. **只想改当前业务项目**
2. **想把改动作为默认模板，后续所有项目都用**

---

## 2. 只改当前业务项目（推荐日常方式）

请改你业务项目里的这两个位置：

1. `docs/<你的PRD目录>/templates/`
2. `.agents/skills/create-prd/references/templates/`

说明：
- `docs/.../templates/` 是项目侧文档模板副本，便于产品同学直接查看。
- `.agents/skills/create-prd/references/templates/` 是 Agent 实际生成时优先使用的模板来源（在该项目内）。

建议做法：
1. 先改 `.agents/skills/create-prd/references/templates/*.md`（核心生效点）。
2. 再同步一份到 `docs/.../templates/*.md`（方便团队查阅与评审）。

---

## 3. 改成“默认模板”（影响后续新安装项目）

请改当前这个仓库里的：

```text
references/templates/
```

改完后建议执行：

```bash
python3 scripts/build.py
```

然后再重新安装到目标项目（或在目标项目里选择“重装 Skill”）。

---

## 4. 常改文件对照表

| 目的 | 文件 |
|---|---|
| 页面 PRD 主体结构 | `references/templates/page-prd-template.md` |
| 变更记录格式 | `references/templates/page-changelog-template.md` |
| 功能清单字段 | `references/templates/feature-list-template.md` |
| 路由清单字段 | `references/templates/route-inventory-template.md` |
| 项目级 Agent 规则 | `references/templates/project-agents-template.md` |

---

## 5. 改完后怎么验证生效

在业务项目执行（注意 `--prd-root` 要和你的目录一致）：

```bash
python .agents/skills/create-prd/scripts/prdctl.py sync . --from-code --prd-root docs/product
python .agents/skills/create-prd/scripts/prdctl.py audit . --level strict --prd-root docs/product
```

你应该看到：
1. 新生成或更新的文档采用了你的新模板结构。
2. 审计报告没有新增结构性错误。

---

## 6. 避坑建议

1. 不要只改 `docs/.../templates` 而不改 `.agents/skills/.../references/templates`，否则 Agent 可能仍按旧模板生成。
2. 模板新增字段后，同步更新 `AGENTS.md` 里的规则描述，避免执行逻辑和模板字段脱节。
3. 重要模板改动建议先在测试项目验证，再推到正式项目。
